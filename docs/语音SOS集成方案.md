# 语音SOS集成方案

## 一句话概括

老人喊"救命/我摔倒了" → ASR Pro 识别 → 触发 ESP32 → SIM7600 自动打电话 + 云告警 → 小程序显示语音SOS告警

---

## 架构

```
老人喊"救命"
    │
    ▼
ASR Pro 2.0（离线语音识别，不需要联网）
    │ 识别到关键词 → 输出高电平
    ▼
ESP32（检测到 GPIO11 高电平 = 等同SOS按钮被按下）
    │
    ├── 蜂鸣器响（本地报警）
    ├── SIM7600 发短信 + 打电话（备用通道）
    └── WiFi HTTP → 云函数 → alerts集合（类型: voice_sos）
            │
            ▼
        小程序家人端
            │
            ├── 告警卡片："🔊 语音求救触发"
            └── 位置查看（如有GPS）
```

## 硬件接线（2根线）

```
ASR Pro 2.0
  ├── 5V      → ESP32 5V（取电）
  ├── GND     → ESP32 GND
  ├── 喇叭+   → 自带小喇叭（ASR Pro语音播报用）
  └── IO/OUT  → ESP32 GPIO11（跟物理按钮并联）
```

**说明：** ASR Pro 输出脚接到 ESP32 的 SOS 按钮引脚上。不管老人按物理按钮还是喊救命，ESP32 看到的都是 GPIO11 变低电平，走同一个告警流程。**代码一行不用改。**

## 组员配置 ASR Pro（天问Block，拖拽编程）

1. 下载天问 Block IDE（免费，图形化编程）
2. 选 ASR Pro 2.0 → 串口连接
3. 拖拽以下模块：

```
唤醒词: "小安小安"
  └── 命令词1: "救命"       → 输出IO拉高，持续5秒
  └── 命令词2: "我摔倒了"   → 输出IO拉高，持续5秒
  └── 命令词3: "快来帮我"   → 输出IO拉高，持续5秒
  └── 命令词4: "小安小安"   → 播报："我在呢，已通知家人"
```

4. 点编译 → 下载到 ASR Pro

**组员的工作量：拖4个模块，10分钟搞定。**

## 小程序改动（你这边）

告警列表新增语音SOS类型：

```javascript
// alerts.wxml 告警类型显示
item.type === 'voice_sos' ? '🔊 语音求救' :
item.type === 'fall' ? '🚨 跌倒告警' : ...
```

其他不用改。因为是 GPI11 同一个引脚，ESP32 固件分不清是按钮还是语音触发的，统一按 SOS 处理。但在云函数 `deviceDataIngest` 里我们可以加判断——如果 SOS 触发但持续时间超过 5 秒（ASR Pro 的配置），标记为 `voice_sos`。

## 答辩亮点

"我们的产品支持两种方式触发求救：物理按钮和语音呼救。老人倒地无法移动时，只需要喊'救命'，系统就能自动通知家人并拨打电话。语音识别用的是 ASR Pro 离线芯片——不需要联网，断网也能用。这是我们为极端情况做的冗余设计。"
