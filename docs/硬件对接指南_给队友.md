## 二、接线图

```
ESP32 引脚         模块
──────────────────────────
GPIO8 (SDA)  ─── MPU6050 SDA ─── MAX30102 SDA   (共用I2C)
GPIO9 (SCL)  ─── MPU6050 SCL ─── MAX30102 SCL   (共用I2C)
GPIO4        ─── DS18B20 DATA
GPIO10       ─── 蜂鸣器 VCC
GPIO11       ─── 按钮 (另一端接 GND)
GPIO16 (RX2) ─── SIM7600 TXD
GPIO17 (TX2) ─── SIM7600 RXD
GPIO18 (RX)  ─── GPS TX
GPIO19 (TX)  ─── GPS RX
5V / GND     ─── 电池盒

所有模块 VCC 接 3.3V 或 5V（看具体模块要求）
所有模块 GND 接 ESP32 GND
```

## 三、改配置（只需要改 2 行）

打开 `sketch_may02a/config.h`，找到第 6-7 行：

```c
#define WIFI_SSID "神都会偏爱"        // ← 改成你的 WiFi 名
#define WIFI_PASSWORD "051024laopo"   // ← 改成你的 WiFi 密码
```

其他不用动（云函数地址、API Key、手机号已经配好）。

**注意：ESP32 只支持 2.4GHz WiFi，手机热点默认就是 2.4GHz。**

## 四、烧录步骤

### 1. 安装 Arduino IDE
https://www.arduino.cc/en/software （下载 2.x 版本）

### 2. 装 ESP32 开发板支持
- File → Preferences → Additional boards manager URLs 粘贴：
  ```
  https://espressif.github.io/arduino-esp32/package_esp32_index.json
  ```
- Tools → Board → Boards Manager → 搜 `esp32` → 安装 **esp32 by Espressif Systems**

### 3. 安装 4 个库
Tools → Manage Libraries → 搜索并安装：
- `MAX30105` by SparkFun
- `OneWire` by Jim Studt
- `DallasTemperature` by Miles Burton
- `TinyGPSPlus` by Mikal Hart

### 4. 烧录
- 用 USB 数据线把 ESP32 连电脑
- Arduino IDE: File → Open → 选 `sketch_may02a/sketch_may02a.ino`
- Tools → Board → esp32 → **ESP32 Dev Module**
- Tools → Port → 选 COMxx（有 ESP32 字样的）
- 点左上角 **→ 箭头**（Upload）

### 5. 看结果
上传完后：Tools → Serial Monitor → 右下角选 **115200** 波特率

成功的话会看到：
```
[WiFi] Connected! IP: 192.168.xxx.xxx
[HTTP] Uploading data... OK (200)
[DATA] BPM=72 SpO2=98% Temp=36.5°C Acc=1.02g SOS=NO WiFi=OK
```

**看到 `OK (200)` 就说明数据已经成功传到云端了！**

## 五、项目和云端的连接

```
ESP32 硬件                   微信云开发                 小程序
┌──────────┐   WiFi/HTTP    ┌──────────────┐        ┌──────────┐
│ 传感器数据 │ ────────────▶ │ healthData表  │ ◀───── │ 家人端    │
│ 跌倒检测   │   每5秒上传    │ alerts 告警   │        │ 老人端    │
│ SOS 按钮  │              │              │        │          │
│ SIM7600  │  备用(短信)    │ 云函数处理    │        │ AI分析    │
└──────────┘              └──────────────┘        └──────────┘
```

**数据流向**：
1. ESP32 采集传感器 → 整理成 JSON → WiFi HTTP POST 到云函数
2. 云函数存数据到数据库 + 检测异常
3. 紧急情况：ESP32 本地蜂鸣器响 + SIM7600 发短信打电话

## 七、常见问题

**Q: ESP32 连不上 WiFi？**
A: 确认热点是 2.4GHz（不是 5GHz）；检查 SSID 和密码拼写；离热点近一点。

**Q: 上传失败 `Error 2`？**
A: 按住 ESP32 上的 BOOT 按钮再点上传，等开始上传了松手。

**Q: MAX30102 读数为 0？**
A: 手指紧贴传感器，等 5-10 秒稳定。

**Q: SIM7600 没反应？**
A: 确认插了 SIM 卡，确认天线接好，确认供电够（SIM7600 吃电，电池不要用太小的）。

**Q: 串口看到 `FAIL` 而不是 `OK`？**
A: 检查云函数 `deviceDataIngest` 是否在云开发控制台部署了；检查 config.h 里的 CLOUD_FUNC_URL 是否正确。

## 文件清单

给你的 `sketch_may02a/` 文件夹里有：
```
sketch_may02a/
├── sketch_may02a.ino    ← 用 Arduino IDE 打开这个
├── config.h             ← 你只需要改这里面的 WiFi 名和密码
├── sensors.h            ← 传感器驱动（不用管）
├── fall_detect.h        ← 跌倒检测算法（不用管）
├── wifi_manager.h       ← WiFi 连接（不用管）
├── http_client.h        ← 数据上传（不用管）
├── sim7600.h            ← SIM7600 打电话（不用管）
├── gps.h                ← GPS 解析（不用管）
└── buzzer.h             ← 蜂鸣器（不用管）
```

改完 `config.h` 里的 WiFi 名和密码后，打开 `.ino` 文件，点上传就行。
