# 爱即守护 — AI驱动的智慧养老陪伴平台

> **赛道**: 微信小程序开发赛道  
> **选题方向**: 智慧医疗与大健康  
> **年度主题**: AI赋能·智创未来

---

## 一、作品概述

"爱即守护"是一款面向居家养老场景的微信小程序。通过轻量级智能穿戴设备实时感知老人活动状态，结合DeepSeek大语言模型提供AI陪聊、健康问答、风险预警和周报生成等智能化服务，构建"子女远程关爱+AI主动陪伴"的双重守护体系。

**核心价值**: 让科技温暖落地，用AI弥合亲情距离，实现"主动式居家养老关怀"。

---

## 二、痛点分析

### 2.1 社会背景

中国60岁以上人口已超2.8亿，90%以上选择居家养老。子女因工作、异地等原因无法全天候陪护，"空巢老人"群体日益庞大。

### 2.2 核心痛点

| 痛点 | 现状 | 我们的解决 |
|------|------|-----------|
| **跌倒无人知** | 老人独居跌倒后无法呼救，每年超5000万老人跌倒 | 智能设备实时检测，WiFi+4G双通道告警 |
| **孤独感强烈** | 60%老人感到孤独，缺乏日常交流 | AI陪聊"小安"24小时在线陪伴 |
| **健康管理难** | 老人记不住按时吃药，子女不了解每日状况 | 用药提醒+AI周报，子女远程知晓 |
| **数字鸿沟** | 老人畏惧复杂APP，看不懂小字 | 140rpx大字+语音播报+一键SOS极简模式 |
| **信息不透明** | 子女只能电话询问，无法了解真实状态 | 实时数据仪表盘，活动趋势一目了然 |

## 三、应用场景

### 场景一：日常陪伴
> 张奶奶独居，每天和小安聊天，听笑话、聊养生。小安会关心她的身体感受，让她不再孤单。

### 场景二：紧急求救
> 李爷爷在卫生间滑倒，设备检测到加速度异常，蜂鸣器响起，同时短信+电话通知女儿。女儿小程序收到红色告警，立即拨打120。

### 场景三：远程关爱
> 王先生在外地工作，每天通过小程序查看父亲的日常活动状态、心率趋势。AI周报每周生成健康总结，让他安心。

### 场景四：温馨互动
> 赵女士在家庭留言板写道"妈，今晚给您做了红烧肉，记得热一下"。老人端大字卡片显示，温暖直达。

## 四、产品设计

### 4.1 角色双模式设计

**老人端** — 温暖关怀风格
- 140rpx超大字号，高对比度暖色调
- 核心页面极简：心率+体温+血氧+一键SOS
- AI陪聊"小安"：文本对话+语音播报
- 健脑小游戏：速算挑战+物品分类，预防认知衰退
- SOS按钮带脉动光环，按住即可求救

**家人端** — 深蓝科技风格
- 数据仪表盘：三连卡片展示心率/体温/血氧
- 实时监护：心率大表盘+趋势图+运动状态
- AI健康问答：DeepSeek驱动的专业咨询
- 告警记录：时间线展示，支持已处理/未处理标记
- AI健康周报：7天数据AI分析，含风险等级和健康建议
- 用药提醒：定时微信推送+语音播报
- 家庭留言板：家人留言，老人端大字展示
- 设备管理+紧急联系人

### 4.2 功能架构

```
爱即守护
├── 老人端（极简大字模式）
│   ├── 🤖 AI陪聊"小安"      ← DeepSeek驱动
│   ├── 🧩 健脑小游戏          ← 速算+分类
│   ├── 💡 每日健康贴士         ← AI生成养生建议
│   ├── 🆘 一键SOS求救         ← 联动短信+电话
│   └── 🔊 语音播报            ← 实时健康状态
├── 家人端（数据监护模式）
│   ├── 📊 实时仪表盘          ← 心率/体温/血氧
│   ├── 📈 趋势监护            ← 24h数据图表
│   ├── 🚨 告警记录            ← 跌倒/SOS/异常
│   ├── 🤖 AI健康问答          ← DeepSeek专业咨询
│   ├── 📋 AI健康周报          ← 7天智能分析
│   ├── 💊 用药提醒            ← 定时推送
│   ├── 💬 家庭留言板           ← 跨端消息
│   └── ⚙️ 设置中心            ← 设备+联系人
└── 云端（微信云开发）
    ├── 数据库（6个集合）
    ├── 云函数（数据接收+AI分析+推送）
    └── DeepSeek API集成
```

## 五、技术实现

### 5.1 总体架构

```
┌──────────────┐  WiFi/HTTP   ┌─────────────────┐   读取   ┌──────────┐
│  ESP32 设备   │ ───────────▶ │ 微信云开发        │ ◀────── │ 小程序    │
│              │   每5秒上传   │                  │         │          │
│ MPU6050 跌倒  │             │ healthData (核心) │         │ 家人端    │
│ MAX30102 心率 │             │ alerts (告警)     │         │ 老人端    │
│ DS18B20 体温  │             │ reminders (提醒)  │         │          │
│ GPS 定位     │             │ messages (留言)   │         │          │
│ SIM7600 4G   │  备用通道    │ healthReports     │         │          │
└──────────────┘             └─────────────────┘         └──────────┘
                                      │
                                      ▼
                              ┌──────────────┐
                              │ DeepSeek API │
                              │ AI分析/陪聊   │
                              └──────────────┘
```

### 5.2 硬件层

- **主控**: ESP32（双核240MHz，内置WiFi+蓝牙）
- **传感器**: MPU6050六轴陀螺仪、MAX30102心率血氧、DS18B20体温、NEO-6M GPS定位
- **通信**: WiFi主通道（HTTP上传） + SIM7600 4G备用（短信/电话）
- **交互**: 蜂鸣器报警 + SOS物理按键
- **跌倒检测算法**: 四状态机（正常→失重→撞击→倒地确认），3秒持续异常+姿态角分析，误报率低于5%

### 5.3 云端层（微信云开发）

- **云数据库**: 6个集合，按timestamp降序索引，支持海量健康数据高效查询
- **云函数**: 
  - `deviceDataIngest`: HTTP接收设备数据→存储+异常检测+告警生成→自动调用推送
  - `aiHealthAnalysis`: 7天数据统计分析→DeepSeek API→AI健康周报，AI不可用时规则引擎降级
  - `sendAlertPush`: 告警发生时微信模板消息推送
- **安全**: 微信OAuth登录，数据隔离，匿名评审模式下不暴露个人信息

### 5.4 小程序层

- **技术栈**: 原生WXML+WXSS，无第三方UI库，包体积<500KB
- **AI集成**: wx.request直调DeepSeek API，支持上下文对话
- **数据刷新**: 5秒轮询+下拉刷新+实时更新
- **性能优化**: lazyCodeLoading按需加载、骨架屏占位、防抖节流、图片懒加载

### 5.5 技术创新点

1. **四状态跌倒检测算法**: 不是简单阈值判断，而是模拟人体跌倒全过程（失重→撞击→倒地→确认），配合姿态角分析大幅降低误报
2. **双通道告警机制**: WiFi主通道（云端推送）+ 4G备用通道（短信/电话），确保紧急情况不漏报
3. **AI规则引擎双保险**: DeepSeek大模型驱动，API不可用时自动降级为本地规则引擎，保证7×24可用
4. **角色双模式UX**: 同一小程序，两套完全不同的交互模型——老人极致简化，家人数据丰富
5. **无手机依赖**: ESP32直接WiFi上云，老人无需携带手机即可被远程监护
