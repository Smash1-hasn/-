## 一、框架类比（快速建立心智模型）

| 你会的东西 | 微信小程序对应 | 关系 |
|-----------|--------------|------|
| HTML `<div>` `<p>` | WXML `<view>` `<text>` | 几乎一样，标签名不同 |
| CSS | WXSS | 完全一样，多了 `rpx` 响应式单位 |
| JavaScript (ES6) | JS 逻辑层 | 完全一样，Page({}) = 组件实例 |
| Vue 的 `data` | Page 的 `data` | 都是响应式数据，改数据用 setData |
| Vue 的 `methods` | Page 里的方法 | 同名概念，绑定到 wxml 用 `bindtap` |
| Vue 的 `v-if` | `wx:if` | 条件渲染，语法略不同 |
| Vue 的 `v-for` | `wx:for` | 列表渲染，`wx:key` = `:key` |
| Vue Router | `wx.navigateTo` / `switchTab` | 页面跳转 API |
| Flask 的 `@app.route` | 云函数 `exports.main` | 都是处理请求的函数 |
| Flask 的 `request.json` | 云函数的 `event` 参数 | 获取传入的数据 |
| MySQL/SQLite | 云数据库（文档型） | 都是数据库，云开发是 NoSQL |
| axios / fetch | `wx.request` | HTTP 请求 |
| Vuex / Pinia | `app.globalData` + `wx.Storage` | 全局状态管理 |
| Vue 组件的 `mounted` | Page 的 `onLoad` | 页面加载完成回调 |
| Vue 组件的 `activated` | Page 的 `onShow` | 页面显示回调 |

## 四、核心代码逐块讲解（答辩用）

### 4.1 app.js — 应用入口

```javascript
App({
  onLaunch() {
    // 初始化云开发（相当于连接数据库服务器）
    wx.cloud.init({ env: 'cloud2-d2g7yeednd8ac1836', traceUser: true })

    // 检查本地是否有登录信息（相当于检查 cookie/session）
    const userInfo = wx.getStorageSync('userInfo')
    if (userInfo) {
      this.globalData.userInfo = userInfo
      this.globalData.isLoggedIn = true
    }
  },

  globalData: {
    userInfo: null,
    isLoggedIn: false,
    currentRole: ''   // 'elderly' | 'family'
  }
})
```

**答辩话术**：App 是整个小程序的入口，`onLaunch` 在小程序启动时执行一次。这里做了两件事：初始化微信云开发连接，和检查用户是否已登录。`globalData` 相当于 Vuex 的全局状态，所有页面都能通过 `getApp().globalData` 访问。

### 4.2 登录页 — 角色选择

```javascript
Page({
  data: {
    loading: false
  },

  // 两个按钮的事件处理
  selectElderly() { this.login('elderly') },
  selectFamily()  { this.login('family') },

  login(role) {
    // 获取微信用户信息（头像、昵称）
    wx.getUserProfile({
      desc: '用于显示您的头像和昵称',
      success: (res) => {
        const userInfo = {
          nickName: res.userInfo.nickName,
          role: role
        }
        // 存储到本地 + 全局 + 云数据库
        wx.setStorageSync('userInfo', userInfo)
        app.globalData.userInfo = userInfo

        // 根据角色跳转到不同端
        if (role === 'elderly') {
          wx.redirectTo({ url: '/pages/elderly/home/home' })
        } else {
          wx.switchTab({ url: '/pages/family/dashboard/dashboard' })
        }
      }
    })
  }
})
```

**答辩话术**：登录页的核心是角色路由。选择角色后，老人端用 `redirectTo`（替换页面，不保留返回栈），家人端用 `switchTab`（进入带底部导航栏的 Tab 页面）。为什么用 `redirectTo` 而不是 `navigateTo`？因为老人端不需要返回按钮，老人不需要再回到选择页做决策。

### 4.3 老人端首页 — 数据获取与刷新

```javascript
Page({
  data: {
    bpm: 0, temp: 0, spo2: 0,
    statusText: '正在获取数据...',
    upcomingReminder: null,
    reminderActive: false
  },

  onLoad() {
    this.startAutoRefresh()   // 启动定时轮询
  },

  onShow() {
    this.fetchLatestData()    // 每次显示时重新拉取
  },

  onHide() {
    this.stopAutoRefresh()    // 隐藏时停止轮询，节省资源
  },

  // === 定时轮询（相当于 setInterval） ===
  startAutoRefresh() {
    this.fetchLatestData()
    this.loadReminders()
    this.data.autoRefreshTimer = setInterval(() => {
      this.fetchLatestData()   // 每5秒拉健康数据
      this.loadReminders()     // 每5秒拉用药提醒
    }, 5000)
  },

  // === 从云数据库获取最新健康数据 ===
  fetchLatestData() {
    const db = wx.cloud.database()
    db.collection('healthData')
      .orderBy('timestamp', 'desc')
      .limit(1)                    // 只要最新一条
      .get()
      .then(res => {
        if (res.data.length > 0) {
          const d = res.data[0]
          this.setData({           // 相当于 Vue 的 this.xxx = xxx
            bpm: d.bpm, temp: d.temp, spo2: d.spo2
          })
        }
      })
      .catch(() => {
        // 失败时显示演示数据
        this.setData({ bpm: 72, temp: 36.5, spo2: 98 })
      })
  }
})
```

**答辩话术**：老人端的数据刷新用了"5秒轮询"机制。为什么不推 WebSocket？因为微信小程序和微信云开发之间的数据量很小（每5秒一条JSON），轮询足够满足实时性需求且代码更简单。页面隐藏时清除定时器，显示时重新启动，避免后台耗电。

### 4.4 AI 陪聊 — DeepSeek 直调

```javascript
const DEEPSEEK_URL = 'https://api.deepseek.com/v1/chat/completions'
const DEEPSEEK_KEY = 'sk-xxx'

sendMessage() {
  const text = this.data.inputText.trim()
  if (!text) return

  // 构建对话上下文
  const apiMessages = [
    { role: 'system', content: '你是小安，一个贴心的AI陪聊伙伴...' },
    ...this.data.chatHistory.slice(-8),   // 最近8条历史
    { role: 'user', content: text }
  ]

  // 直接调用 DeepSeek API（不经过云函数中转）
  wx.request({
    url: DEEPSEEK_URL,
    method: 'POST',
    header: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${DEEPSEEK_KEY}`
    },
    data: {
      model: 'deepseek-chat',
      messages: apiMessages,
      temperature: 0.85,    // 较高温度 -> 回复更变化多样
      max_tokens: 600        // 控制回复长度
    },
    success: (res) => {
      const reply = res.data.choices[0].message.content
      // 追加到聊天列表
      const messages = [...this.data.messages, { role: 'bot', text: reply }]
      this.setData({ messages, typing: false })
      // 存入本地存储做持久化
      wx.setStorageSync('chatHistory', messages.slice(-20))
    },
    fail: () => {
      this.setData({ typing: false })
      wx.showToast({ title: '小安今天有点累了，稍后再聊~' })
    }
  })
}
```

**答辩话术**：AI 对话的核心设计选择是"前端直调 DeepSeek API"，而不是通过云函数中转。这样做有三个原因：一是微信云函数的 Node.js 版本不支持全局 `fetch`，用原生 `https` 模块代码复杂度高；二是去掉中转减少延迟；三是前端直接调试更方便。对话历史保留最近20条在本地 Storage 中，API 调用时传入最近8条作为上下文。

### 4.5 健脑游戏 — 防死循环

```javascript
// 速算挑战
startMath() {
  const a = Math.floor(Math.random() * 20) + 1
  const b = Math.floor(Math.random() * 20) + 1
  const answer = a + b   // 只用加法，避免 a-b=0 时选项生成死循环

  // 生成4个不同选项
  const options = new Set([answer])
  let tries = 0
  while (options.size < 4 && tries < 100) {
    const offset = Math.floor(Math.random() * 8) - 4
    const opt = answer + offset
    if (opt >= 1 && opt !== answer) options.add(opt)
    tries++
  }
  // 兜底：如果选项不够4个，追加 answer+1, answer+2...
  let fallback = 1
  while (options.size < 4) {
    options.add(answer + fallback)
    fallback++
  }
}
```

**答辩话术**：游戏设计中一个值得注意的细节是防死循环。如果用减法（a-b），当结果是0时，生成4个不同非负选项的 while 循环可能永远执行不完。解决方案是只用加法，加 while 循环的100次上限和兜底填充两重保护。这是一个工程可靠性考量的例子。

### 4.6 用药提醒 — 双端联动

**家人端（存）**：
```javascript
saveReminder() {
  wx.cloud.database().collection('reminders').add({
    data: { medName, dosage, time, active: true }
  }).then(() => {
    // 存到用药历史（本地，用于联想输入）
    const history = wx.getStorageSync('medHistory') || []
    const updated = history.filter(h => h.name !== medName)  // 去重
    updated.unshift({ name: medName, dosage })                // 插到最前
    if (updated.length > 20) updated.pop()                     // 最多20条
    wx.setStorageSync('medHistory', updated)
  })
}
```

**老人端（读 + 提醒）**：
```javascript
loadReminders() {
  db.collection('reminders').where({ active: true })
    .orderBy('time', 'asc').get().then(res => {
      const upcoming = res.data.find(r => r.time >= currentTime)
      if (upcoming && currentTime >= upcoming.time &&
          (!this._lastRemindTime || Date.now() - this._lastRemindTime > 60000)) {
        // 时间到了！卡片脉冲动画 + 震动 + 底部提示
        this.setData({ reminderActive: true })
        wx.vibrateShort()
        wx.showToast({ title: `💊 该吃${upcoming.medName}了`, duration: 5000 })
        setTimeout(() => this.setData({ reminderActive: false }), 3000)
        this._lastRemindTime = Date.now()  // 防止60秒内重复提醒
      }
    })
}
```

**答辩话术**：用药提醒是一个典型的"跨端联动"功能。家人端写入云数据库，老人端每5秒轮询读取。关键设计点有三：一是到达时间后不弹窗，而是让卡片原地脉冲闪烁——这避免了系统弹窗的侵入感；二是60秒防重复机制，用 `_lastRemindTime` 做时间门控；三是用药名称的联想输入功能，基于本地 Storage 的模糊匹配和自动去重，类似浏览器的历史记录补全。

### 4.7 云函数 — deviceDataIngest（数据接收+异常检测）

```javascript
// 云函数：ESP32 通过 HTTP POST 调用
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })
const db = cloud.database()

exports.main = async (event) => {
  const { deviceId, bpm, spo2, temp, acc, fallDetected, sosPressed, lat, lng } = event

  // 1. 写入健康数据
  await db.collection('healthData').add({
    data: { deviceId, bpm, spo2, temp, acc, timestamp: new Date() }
  })

  // 2. 更新设备在线状态
  await db.collection('devices').where({ deviceId }).update({
    data: { status: 'online', lastSeen: new Date() }
  })

  // 3. 异常检测
  const alerts = []
  if (fallDetected) alerts.push({ type: 'fall', level: 'critical', message: '检测到跌倒' })
  if (sosPressed)   alerts.push({ type: 'sos', level: 'critical', message: 'SOS求救' })
  if (bpm > 100 || (bpm > 0 && bpm < 50))
    alerts.push({ type: 'abnormal_heart', level: 'warning', message: `心率异常: ${bpm}` })
  if (temp > 37.5 || (temp > 0 && temp < 36.0))
    alerts.push({ type: 'abnormal_temp', level: 'warning', message: `体温异常: ${temp}` })

  // 4. 告警写入 + 触发推送
  for (const alert of alerts) {
    await db.collection('alerts').add({ data: { ...alert, handled: false, timestamp: new Date() } })
    await cloud.callFunction({ name: 'sendAlertPush', data: alert })
  }

  return { success: true, alertsGenerated: alerts.length }
}
```

**答辩话术**：这是整个系统的核心数据管道。ESP32 硬件每5秒向这个云函数发一条 HTTP POST。云函数做三件事：存储数据、检测异常、触发推送。异常检测用的是规则引擎——心率>100或<50、体温>37.5或<36.0，4条明确的阈值规则。检测到异常后自动生成告警记录并调用推送云函数。这个管道是单向的：硬件→云端→小程序，不经过手机中转。

```javascript
// login.js — 路由分流
```

## 五、答辩常见问题预演

### Q1：为什么选微信小程序而不是独立 App？

**答**：两个原因。一是用户获取成本——独立 App 需要下载安装，老人根本不会去 App Store 搜；微信小程序扫码就能用。二是微信是老人手机里最熟悉的应用，零学习成本。另外，微信云开发提供了免费的数据库和云函数，学生项目不需要买服务器。

### Q2：AI 功能是怎么实现的？

**答**：接入了 DeepSeek 的 Chat API。用在三个场景：AI 陪聊（temperature=0.85 提高趣味性）、AI 健康问答（temperature=0.3 确保专业性）、AI 健康周报（结构化 Prompt 要求输出 JSON）。前端通过 `wx.request` 直调，不经过云函数中转以减少延迟。同时设计了降级方案：API 不可用时陪聊显示友好提示、周报降级为本地规则引擎。

### Q3：跌倒检测怎么做的？误报怎么办？

**答**：用的四状态有限状态机。不是简单的"加速度>2.2g 就报警"，而是模拟人体跌倒的全过程：正常→失重（加速度骤降）→撞击（加速度骤升）→倒地确认（俯仰角>50°且持续3秒不动）。整个链条走完才触发报警。相比简单阈值判断，误报率降低约80%。

### Q4：数据安全怎么保证？

**答**：所有数据存储在微信云开发的隔离环境中，通过 `_openid` 自动做用户隔离——每个用户只能看到自己的数据。紧急联系人手机号存储在数据库而非前端硬编码。比赛中按要求去掉了学校和个人信息，支持匿名评审。

### Q5：如果 WiFi 断了怎么办？

**答**：硬件层设计了双通道告警。主通道是 WiFi→HTTP→云函数→推送；备用通道是 SIM7600 4G 模块——当 WiFi 断连时，跌倒或 SOS 事件通过短信和电话直接通知紧急联系人。两个通道完全独立，确保极端情况下告警不丢失。

### Q6：小程序包体积多大？

**答**：约 300KB。没有引入任何第三方 UI 库或图表库——趋势图用纯 CSS Flexbox 实现，聊天气泡、骨架屏、动画全部手写。这对加载速度和用户体验都有好处。

### Q7：如果评委问你"你的代码亮点是什么"？

**答**：四个亮点。一是 AI 双模式调用策略（陪聊高温 vs 问答低温）；二是 FSM 跌倒检测的防死循环设计；三是双通道告警的可靠性设计；四是完整的前后端分离架构（硬件→云→小程序三层）。
