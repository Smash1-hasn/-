## 总体架构

```
┌──────────────────────────────────────────────────────┐
│  硬件层 (ESP32 穿戴设备)                               │
│  ├ MPU6050  跌倒检测（改进算法）                         │
│  ├ MAX30102 心率/血氧                                   │
│  ├ DS18B20  体温                                        │
│  ├ NEO-6M   GPS定位                                     │
│  ├ SIM7600  紧急电话/SMS（备用通道）                      │
│  ├ 蜂鸣器 + SOS物理按键                                  │
│  └ WiFi → HTTP POST → 云函数                            │
├──────────────────────────────────────────────────────┤
│  云服务层 (微信云开发)                                   │
│  ├ 云数据库（设备数据 / 用户 / 告警）                     │
│  ├ 云函数（数据接收 / 分析 / 推送）                       │
│  ├ 云存储（健康报告）                                    │
│  └ DeepSeek API（AI分析）                               │
├──────────────────────────────────────────────────────┤
│  小程序层                                               │
│  ├ 老人端：极简大字 + 语音播报 + 一键求救                  │
│  └ 子女端：实时仪表盘 + 健康趋势 + AI报告 + 告警推送       │
└──────────────────────────────────────────────────────┘
```

**关键设计**:
- ESP32 直接通过 WiFi 发送数据到云 → 手机不需要在旁边
- SIM7600 仅作为网络断开时的备用告警通道（电话+短信）
- 微信云开发 → 免费、免服务器、与小程序深度集成

## 硬件清单

| 模块 | 作用 | 接口 |
|------|------|------|
| ESP32 | 主控 + WiFi + 蓝牙 | - |
| MPU6050 | 6轴加速度/陀螺仪 跌倒检测 | I2C (SDA:8, SCL:9) |
| MAX30102 | 心率+血氧 | I2C |
| DS18B20 | 体温 | OneWire (Pin 4) |
| SIM7600 | 4G电话/SMS 备用告警 | Serial1 (RX:16, TX:17) |
| 蜂鸣器 | 本地告警 | Pin 10 |
| SOS按钮 | 物理呼救 | Pin 11 |
| GPS NEO-6M | 实时定位，子女看位置 | Serial2 (RX:18, TX:19) |

## ESP32 固件结构（模块化C++）

```
sketch_may02a/
├── sketch_may02a.ino       # 主入口 setup() + loop()
├── config.h                 # WiFi、手机号、云函数URL、DeepSeek Key
├── sensors.h                # MPU6050、MAX30102、DS18B20、GPS 读取
├── fall_detect.h            # 改进跌倒检测（3秒确认 + 姿态角）
├── wifi_manager.h           # WiFi连接 + 自动重连
├── http_client.h            # HTTP POST 上传到云函数
├── sim7600.h                # SIM7600 电话 + 短信
├── gps.h                    # NEO-6M GPS 解析
└── buzzer.h                 # 蜂鸣器控制
```

**改进的跌倒检测**: 瞬时加速度 + 姿态角变化率 → 3秒持续异常才报警 → 区分跌倒/弯腰/坐下

**数据上报** (每5秒):
```json
{
  "deviceId": "ESP32_001", "bpm": 72, "spo2": 98, "temp": 36.5,
  "acc": 1.05, "gyroX": 0.1, "gyroY": 0.2, "gyroZ": 0.1,
  "fallDetected": false, "sosPressed": false,
  "battery": 85, "rssi": -45, "lat": 38.9136, "lng": 121.6147
}
```

## 微信云开发

### 云数据库集合

| 集合 | 说明 |
|------|------|
| users | 用户信息，role: "elderly" / "family" |
| devices | 设备绑定，状态 online/offline |
| healthData | 健康数据（核心），每条传感器上报记录 |
| alerts | 告警记录，type: fall/sos/abnormal_heart/abnormal_temp |
| healthReports | AI健康周报 |
| emergencyContacts | 紧急联系人 |

### 云函数

| 云函数 | 触发 | 功能 |
|--------|------|------|
| deviceDataIngest | HTTP（ESP32调用） | 接收数据 → 存储 → 异常检测 |
| aiHealthAnalysis | 小程序调用 | 调用DeepSeek API分析 |
| generateWeeklyReport | 定时触发 | 生成一周健康总结 |
| sendAlertPush | 数据库触发器 | 告警时微信推送 |
| userManage | 小程序调用 | 用户注册、设备绑定 |

## AI 功能（比赛核心竞争力）

1. **AI跌倒风险预测** — 分析7天加速度数据，DeepSeek评估步态稳定性，输出风险等级
2. **AI健康异常检测** — 心率+体温组合分析，规则引擎+AI辅助判断
3. **AI健康周报** — 自动生成周报：统计、趋势、异常汇总、AI健康建议
4. **语音播报**（老人端）— "您当前心率72，体温36.5，一切正常"
