# 安心守护 — 从零开始操作指南

> 你不需要有编程经验，跟着做就行。预计1-2小时完成全部配置。

---

## 第一步：开通微信云开发（10分钟）

1. 打开 **微信开发者工具**（没有就下载安装 https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html）
2. 用你的微信扫码登录
3. 导入项目：选择 `d:\移动开发赛\miniprogram` 文件夹
4. AppID 填 `wx87be231a021d8486`
5. 点击工具栏的 **「云开发」** 按钮
6. 点击「开通」，环境名称填 `guardian-cloud`
7. 开通后，你会看到 **环境ID**（类似 `guardian-cloud-xxx`）
8. **记下这个环境ID！**

---

## 第二步：创建数据库（5分钟）

1. 云开发控制台 → 数据库
2. 创建以下 6 个集合（点击「添加集合」）：
   - `users`
   - `devices`
   - `healthData`
   - `alerts`
   - `healthReports`
   - `emergencyContacts`
3. 不需要添加任何数据，空集合即可

---

## 第三步：部署云函数（15分钟）

1. 在微信开发者工具左侧文件列表中，找到 `cloudfunctions` 文件夹
2. **右键** `deviceDataIngest` → 「上传并部署：云端安装依赖」
3. 等待上传完成（看控制台日志）
4. 同样右键 `aiHealthAnalysis` → 「上传并部署：云端安装依赖」
5. 同样右键 `sendAlertPush` → 「上传并部署：云端安装依赖」

---

## 第四步：配置 HTTP 触发器（5分钟）

1. 云开发控制台 → 云函数 → 点击 `deviceDataIngest`
2. 点击「触发器」标签 → 「添加触发器」
3. 选择「HTTP 触发器」
4. 复制生成的 **HTTP 触发 URL**（类似 `https://xxx.service.tcloudbase.com/deviceDataIngest`）

---

## 第五步：修改配置文件（5分钟）

打开 `d:\移动开发赛\miniprogram\app.js`，找到这行：
```javascript
env: 'guardian-cloud',
```
把 `guardian-cloud` 替换成你第一步记下的 **环境ID**（如 `guardian-cloud-abc123`）

打开 `d:\移动开发赛\sketch_may02a\config.h`，修改：
```c
// 1. WiFi — 改成你手机的热点名和密码
#define WIFI_SSID "你的手机热点名"
#define WIFI_PASSWORD "你的热点密码"

// 2. 云函数URL — 粘贴第四步复制的URL
#define CLOUD_FUNC_URL "https://你的环境ID.service.tcloudbase.com/deviceDataIngest"

// 3. DeepSeek API Key — 把你的Key填在这里
#define DEEPSEEK_API_KEY "你的DeepSeek_API_Key"

// 4. 紧急联系人手机号
#define EMERGENCY_PHONE "18009800204"
```

## 第六步：上传ESP32固件（10分钟）

1. 打开 Arduino IDE
2. 安装需要的库（工具 → 管理库）：
   - `MAX30105` by SparkFun
   - `OneWire`
   - `DallasTemperature`
   - `TinyGPSPlus` by Mikal Hart
3. 打开 `d:\移动开发赛\sketch_may02a\sketch_may02a.ino`
4. 选择开发板：`ESP32 Dev Module`
5. 选择正确的串口
6. 点击「上传」
7. 打开串口监视器（波特率 115200），看到以下输出说明成功：
   ```
   [WiFi] Connected! IP: 192.168.x.x
   [HTTP] Uploading data... OK
   ```

## 常见问题

**Q: 小程序编译报错「找不到xxx页面」**
A: 确认 `app.json` 中的 pages 路径和实际文件路径一致

**Q: 云函数部署失败**
A: 右键云函数文件夹 → 「云函数本地调试」→ 检查是否有语法错误

**Q: ESP32连不上WiFi**
A: 手机热点确保是 2.4GHz（ESP32不支持5GHz）；检查 SSID 和密码有没有写错

**Q: ESP32连上WiFi但数据上传失败**
A: 检查 `config.h` 中的 `CLOUD_FUNC_URL` 是否正确；确认云函数已部署且HTTP触发器已添加

**Q: Tab栏没有图标**
A: 这是正常的，需要准备8张81x81的PNG图标放到 `images/` 文件夹。暂时不配图标不影响使用
