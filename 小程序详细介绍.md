## 一、项目概述

### 1.1 项目定位

"爱即守护"是一款面向居家养老场景的微信小程序。产品通过ESP32智能穿戴设备实时采集老人活动数据，结合DeepSeek大语言模型提供AI陪聊、健康问答、风险预警及健康周报等智能服务，构建"子女远程关爱 + AI主动陪伴"的双重守护体系。

项目旨在解决居家养老场景中三大核心矛盾：**老人独居安全难以保障**、**子女异地无法实时了解状况**、**老年人存在数字鸿沟畏惧复杂APP**。

### 1.2 目标用户

| 用户角色 | 特征 | 核心诉求 |
|---------|------|---------|
| 老年用户（被监护人） | 70岁以上，视力下降，不熟悉智能设备，可能独居 | 操作简单、看得清、有陪伴感、紧急情况能求助 |
| 子女用户（监护人） | 异地工作，无法全天候陪护，担心父母安全 | 远程了解状态、异常及时通知、获得专业健康参考 |

### 1.3 核心差异化

- **AI主动陪伴而非被动监测**：不是冷冰冰的数据面板，而是有温度的AI陪聊、健康周报
- **老人端极致简化**：字体140rpx，全页面仅需操作心率卡片和SOS按钮
- **双通道高可靠告警**：WiFi主通道+4G备用通道，确保紧急情况零漏报
- **无手机依赖**：设备直接WiFi上云，老人无需携带手机

## 三、登录页详解

### 3.1 设计理念

登录页是小程序的"第一印象"。设计上采用**双色双圆Logo**作为核心视觉锚点——左侧暖橙圆代表"老人守护"，右侧冷蓝圆代表"家人关爱"，两圆之间的渐变线暗示"AI连接两代人"的产品理念。

### 3.2 视觉元素

- **背景**：冷蓝→暖橙→纯白的170°对角线渐变，暗示产品从"科技"走向"温度"
- **装饰圆**：4个不同大小和位置的半透明径向渐变圆，营造空间层次感
- **Logo双圆**：120rpx圆形，带box-shadow立体阴影，触控有缩放动效
- **应用名**：56rpx，900字重，CSS渐变文字（蓝→橙），字间距6rpx
- **角色按钮**：白色卡片，左侧6rpx彩色边框（老人端暖珊瑚色/家人端信赖蓝），按压缩放动效

### 3.3 交互逻辑

- 点击「我是长辈」→ 弹出微信授权（可选拒绝）→ 存储用户信息到本地 → 保存到云数据库users集合 → redirectTo跳转老人端首页
- 点击「我是家人」→ 同上流程 → switchTab跳转家人端仪表盘
- 用户拒绝授权时使用默认昵称，不影响功能使用
- 已登录用户再次进入自动跳过登录页，按角色直接跳转

### 3.4 技术实现要点

```javascript
// 角色选择核心逻辑

login(role) {
  // 1. 微信授权获取昵称头像
  // 2. 写入本地Storage + 全局globalData
  // 3. 异步写入云数据库users集合
  // 4. navigateByRole跳转对应首页
}
```

## 四、老人端详解

### 4.1 老人首页 (elderly/home)

#### 4.1.1 设计理念

老人端遵循"极简主义+极致关怀"设计原则。全页面信息密度极低——仅展示心率、体温、血氧三个核心数据，配以超大字号的SOS按钮。配色采用暖金色调（#FFF8F0→#FFFBF7渐变背景），让老人感到温暖而非冰冷的医疗感。

#### 4.1.2 页面布局

```
┌─────────────────────────────────┐
│  ← 切换          14:30      🔋  │ ← 顶部状态栏
├─────────────────────────────────┤

├─────────────────────────────────┤
│         心 率                   │

│      点击刷新                   │   140rpx超大字号
├─────────────────────────────────┤

│  🌡️ 36.5°C │  💧 98%           │ ← 体温+血氧迷你卡
│    体温     │    血氧            │
├────────────┴────────────────────┤
│                                 │
│         ╭─────────╮             │
│         │  S O S  │             │ ← 300rpx SOS按钮
│         │ 按住求救  │             │   带脉动光环动画
│         ╰─────────╯             │
│                                 │
├─────────────────────────────────┤
│   🔊 点击播报健康状况            │ ← 语音播报条
└─────────────────────────────────┘
```

#### 4.1.3 核心功能

**心率显示**
- 字号：140rpx，字重：900
- 红色渐变色（#E53935→#C62828）
- 心跳emoji动画：`animation: heartBeat 0.8s infinite`
- 点击卡片手动刷新数据
- 数据库无数据时显示"--"

**SOS一键求救**
- 300rpx圆形按钮，红色渐变（#FF5252→#B71C1C）
- 外圈脉动光环动画：`@keyframes sosRing` (scale 0.9→1.25, opacity 0.5→0)
- 点击流程：确认弹窗 → 写入alerts集合 → 调用紧急联系人号码 → 震动反馈
- 紧急联系人从云数据库emergencyContacts集合读取，按priority升序取第一个

**健康贴士**
- 12条预置养生建议，随机抽取一条
- 内容覆盖：饮水、运动、午休、饮食、心情、泡脚、晒太阳等
- 弹窗展示，确认即关闭

**语音播报**
- 底部温水色渐变条
- 点击后Toast展示健康状态文本，模拟语音播报
- 文本格式："您当前心率72次，体温36.5度。一切正常。"

#### 4.1.4 数据刷新机制

- onLoad时启动5秒定时轮询
- 每次onShow重新拉取最新数据
- 页面隐藏(onHide)时停止定时器，节省资源
- 数据失败时显示演示数据（心率72、体温36.5、血氧98）

### 4.2 AI陪聊页 (elderly/chat)

#### 4.2.1 功能定位

"小安"是24小时在线的AI陪聊伙伴，基于DeepSeek大语言模型驱动的对话系统。目标是缓解老人孤独感，提供日常情感陪伴。

#### 4.2.2 对话设计

- **系统人设**：温暖贴心的孙辈角色，称呼"您"和"老人家"，语气像唠家常
- **回复风格**：每次3-5句话，有温度有内容，可以讲笑话、聊养生、问身体感受
- **安全边界**：发现老人说身体不舒服时温和提醒联系家人，不推荐具体药物

#### 4.2.3 界面布局

```
┌─────────────────────────────────┐
│  今天天气真好 │ 我有点闷          │ ← 5个快捷话题chips
│  讲个笑话吧   │ 养生小知识        │
├─────────────────────────────────┤
│  🤖 您好！我是小安...            │ ← AI问候
│                                 │
│                    👴 今天有点闷  │ ← 用户消息（右对齐蓝色）
│                                 │
│  🤖 老人家别闷啦～我给您讲个       │ ← AI回复（左对齐白色圆角）
│  笑话吧...                      │
│                                 │
│  🤖 ● ● ●  (打字中动画)         │ ← 等待状态
├─────────────────────────────────┤
│  🎤                  ➤         │ ← 底部输入栏
└─────────────────────────────────┘
```

#### 4.2.4 技术实现

- **通信方式**：小程序前端`wx.request`直调DeepSeek Chat API
- **模型参数**：`deepseek-chat`, temperature=0.85, max_tokens=600
- **上下文管理**：保留最近20条对话存储在本地Storage，API调用时传入最近8条
- **域名配置**：需在微信后台添加`https://api.deepseek.com`到request合法域名
- **降级处理**：API失败时显示友好的离线提示

#### 4.2.5 快捷话题

预设5个话题chips，老人一键点击即可发起对话：
- "今天天气真好" → 展开聊天气和生活
- "我有点闷" → 触发AI讲笑话或趣事
- "讲个笑话吧" → 直接请求幽默内容
- "养生小知识" → 科普简单的养生常识
- "谢谢小安" → 礼貌结束对话

### 4.3 健脑游戏页 (elderly/games)

#### 4.3.1 功能定位

提供轻度认知训练游戏，帮助老人锻炼脑力、预防认知衰退。两个游戏均无需任何说明即可上手。

#### 4.3.2 游戏一：速算挑战

**游戏规则**：显示一道20以内的加法题（a+b=?），从4个选项中选出正确答案。

**交互设计**：
- 题目字号：88rpx，字重900，大间距
- 4个选项按钮：140×100rpx，蓝底白字，被选中时缩放动效
- 正确答案高亮：绿色边框+绿色背景
- 错误答案：显示"答案是XX"提示
- 连续3题以上有🔥连胜标记和额外加分
- 每题作答后需手动点击"下一题→"进入下一局

**分数系统**：
- 每题答对+10分
- 连续3题以上额外+5分
- 最高分存储在本地Storage持久化
- 答错连胜归零

#### 4.3.3 游戏二：物品归类

**游戏规则**：从6个选项中找出不属于指定类别（水果/蔬菜/动物/衣物）的那个。

**交互设计**：
- 类别名以蓝色加粗提示
- 6个选项横向排列，白色圆角卡片
- 选中后反馈正确/错误
- 正确选项绿色高亮

**训练价值**：分类思维是认知功能的重要维度，锻炼语义记忆和逻辑判断。

#### 4.3.4 技术要点

- 全纯前端逻辑，不依赖云函数
- 防死循环设计：while循环100次上限+兜底填充
- 页面卸载(onUnload)时清理定时器
- 返回菜单时重置计分和连胜

#### 5.1.3 智能问候语

根据当前小时自动切换：
- 0-6点："夜深了"
- 6-9点："早上好"
- 9-12点："上午好"
- 12-14点："中午好"
- 14-18点："下午好"
- 18-0点："晚上好"

#### 5.1.4 数据卡片

三连卡片采用玻璃质感设计（`rgba(255,255,255,0.95)` + `backdrop-filter: blur(20rpx)`），数值使用CSS渐变色（心率红色渐变、体温珊瑚渐变、血氧蓝色渐变），56rpx字重800。

#### 5.1.5 AI风险评估

从`healthReports`集合读取最新报告的三个维度：
- **跌倒风险**：低(绿)/中(橙)/高(红)，带圆环图标和颜色区分
- **步态稳定性**：正常/需关注/建议就医
- **心率趋势**：正常/需关注

高风险状态下红色圆环有`animation: pulse 2s infinite`脉动动画。

#### 5.1.6 最近告警

按timestamp降序取最近5条，展示类型图标+消息文本+时间。`critical`级别红点，`warning`级别橙点。

#### 5.1.7 快捷入口

两行四列的快捷功能矩阵，每个入口带emoji图标+文字说明，点击跳转到对应功能页。

### 5.3 告警记录页 (family/alerts)

#### 5.3.1 功能定位

展示所有历史告警记录，支持按类型筛选和标记处理。

#### 5.3.2 筛选标签

| 筛选项 | 说明 | 实现 |
|--------|------|------|
| 全部 | 显示所有告警 | filter='all' |
| 未处理 | 仅显示handled=false的告警 | filter='unhandled' |
| 紧急 | 仅显示level='critical'的告警 | filter='critical' |

WXML中使用`data-filter`属性+`bindtap="filterAlerts"`切换筛选状态。

#### 5.3.3 告警类型

| 类型 | 图标 | 触发条件 |
|------|------|---------|

## 七、云端技术架构

### 7.1 云数据库集合设计

#### healthData（核心集合）
```
{
  deviceId: "ESP32_001",    // 设备ID
  bpm: 72,                  // 心率
  spo2: 98,                 // 血氧
  temp: 36.5,               // 体温
  acc: 1.02,                // 合加速度
  gyroX/Y/Z: 0.1,          // 陀螺仪
  pitch: 12.5,              // 俯仰角
  roll: 3.2,                // 翻滚角
  fallDetected: false,      // 跌倒标记
  sosPressed: false,        // SOS标记
  battery: 85,              // 电量
  rssi: -45,               // WiFi信号强度
  lat: 38.9136, lng: 121.6147,  // GPS坐标
  timestamp: Date           // 时间戳（索引）
}
```

#### alerts（告警集合）
```
{
  deviceId, type, level, message, bpm, temp,
  handled: false,           // 是否已处理
  timestamp: Date           // 时间戳（索引）
}
```

#### 其他集合
- **users**: { _openid, nickName, avatarUrl, role, createTime }
- **devices**: { deviceId, status, rssi, lastSeen, bindTime }
- **healthReports**: { weekStart, weekEnd, summary, aiAnalysis, riskLevel, stepStability, suggestions, avgBpm, avgTemp, alertCount, timestamp }
- **emergencyContacts**: { name, phone, relation, priority, createTime }
- **reminders**: { medName, dosage, time, repeat, active, createTime }
- **messages**: { content, type, fromRole, fromUser, createTime }

### 7.2 云函数详解

#### deviceDataIngest

- **触发方式**：HTTP触发器，ESP32通过HTTP POST调用
- **功能**：接收JSON数据→解析校验→写入healthData→更新设备状态→多维度异常检测
- **异常检测规则**：
  - 跌倒：fallDetected=true → 生成critical级别fall告警
  - SOS：sosPressed=true → 生成critical级别sos告警
  - 心率异常：bpm>100或bpm<50 → 生成warning级别abnormal_heart告警
  - 体温异常：temp>37.5或temp<36.0 → 生成warning级别abnormal_temp告警
- **推送联动**：生成告警后调用sendAlertPush云函数推送微信消息

#### aiHealthAnalysis

- **触发方式**：小程序调用
- **功能流程**：
  1. 计算7天数据统计（均值、范围、异常次数）
  2. 构建专业医疗分析Prompt
  3. 调用DeepSeek API获取AI分析
  4. API失败时降级为本地规则引擎
  5. 返回结构化分析结果
- **规则引擎降级**：
  - 跌倒次数>0或加速度峰值>2.5g → 高风险，建议就医
  - 高加速度>5次或加速度峰值>2.0g或心率异常>10次 → 中风险，需关注
  - 其他 → 低风险，保持现状
  - 每个风险等级有对应的建议模板

#### sendAlertPush

- **触发方式**：被deviceDataIngest调用
- **功能**：通过微信模板消息向绑定的家人用户发送告警推送
- **权限**：需配置openapi.subscribeMessage.send

### 7.3 AI技术方案

#### DeepSeek集成方式

小程序前端通过`wx.request`直接调用DeepSeek Chat API：

```javascript
wx.request({
  url: 'https://api.deepseek.com/v1/chat/completions',
  method: 'POST',
  header: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${API_KEY}`
  },
  data: {
    model: 'deepseek-chat',
    messages: [{ role: 'system', content: SYSTEM_PROMPT }, ...],
    temperature: 0.85,   // 陪聊用高温增加趣味性
    max_tokens: 600
  }
})
```

#### AI应用的三处调用点

| 应用 | temperature | max_tokens | 特点 |
|------|------------|------------|------|
| AI陪聊（老人端） | 0.85 | 600 | 高温增加对话趣味性 |
| AI健康问答（家人端） | 0.3 | 600 | 低温确保专业准确 |
| AI健康周报 | 0.3 | 800 | 结构化JSON输出 |

## 九、性能优化方案

### 9.1 代码层面

- **lazyCodeLoading**：`app.json`配置按需加载，只有访问的页面才下载代码
- **包体积控制**：不引入第三方UI库/图表库，纯原生实现，总体积约300KB
- **setData优化**：增量更新，只传变化的数据字段，避免全量刷新
- **定时器管理**：页面onHide时停止，onShow时恢复，避免后台消耗

### 9.2 数据库层面

- **索引优化**：healthData和alerts按timestamp降序索引
- **查询限制**：告警列表limit(50)，趋势数据limit(288)，历史报告limit(5)
- **按需查询**：每个页面只查询自己需要的数据，不做全表扫描

### 9.3 网络层面

- **数据上传频率**：正常5秒一次，告警时立即发送
- **超时设置**：wx.request timeout 20秒
- **重试机制**：WiFi断开时ESP32自动重连
